ON-IT 앱버전 개발일기 5일차: SwiftUI로 무한 캔버스 제스처 구현하기

2026년 6월 4일
프론트엔드 프로젝트 학습
SwiftUI Canvas Gesture 무한캔버스 좌표변환

5일차에는 ON-IT의 핵심인 캔버스를 앱 안으로 가져왔습니다. 웹에서는 마우스와 브라우저 좌표를 기준으로 생각했다면, iOS에서는 손가락 제스처와 SwiftUI 렌더링 흐름에 맞게 다시 생각해야 했습니다.

캔버스 좌표와 화면 좌표를 분리했다

무한 캔버스에서 가장 헷갈리는 부분은 “노트의 실제 위치”와 “지금 화면에서 보이는 위치”가 다르다는 점입니다. 확대/축소와 이동이 들어가면 이 둘을 섞어 쓰는 순간 드래그와 생성 위치가 틀어집니다.

GeometryReader { geo in
    ZStack {
        DotGrid(scale: liveScale, offset: liveOffset)

        ForEach(vm.notes) { note in
            NoteCardView(note: note, scale: liveScale)
                .position(
                    x: note.positionX * liveScale + liveOffset.width,
                    y: note.positionY * liveScale + liveOffset.height
                )
        }
    }
    .gesture(canvasDragGesture)
    .simultaneousGesture(canvasMagnificationGesture)
}

그래서 노트 데이터에는 캔버스 기준 좌표만 저장하고, 화면에 그릴 때만 `scale`과 `offset`을 곱했습니다. 이 방식이면 확대 상태에서 노트를 움직여도 서버에 저장되는 값은 항상 같은 기준을 유지합니다.

더블탭 생성도 좌표 변환이 핵심이었다

앱에서 노트를 빠르게 만들려면 툴바 버튼을 누르고 위치를 고르는 방식보다, 캔버스를 두 번 탭해서 바로 만드는 편이 자연스럽다고 생각했습니다. 하지만 탭 위치는 화면 좌표이기 때문에 캔버스 좌표로 변환해야 했습니다.

private func canvasPoint(from screenPoint: CGPoint) -> CGPoint {
    CGPoint(
        x: (screenPoint.x - liveOffset.width) / liveScale,
        y: (screenPoint.y - liveOffset.height) / liveScale
    )
}

.onTapGesture(count: 2, coordinateSpace: .local) { location in
    let point = canvasPoint(from: location)
    Task {
        await vm.addNote(
            content: "새 노트",
            x: point.x,
            y: point.y
        )
    }
}

이 수식은 작지만 중요했습니다. 화면에서 100픽셀 움직인 것과 캔버스에서 100픽셀 움직인 것은 확대율에 따라 완전히 다릅니다. 그래서 모든 입력은 캔버스 기준으로 다시 바꾼 뒤 ViewModel에 넘기게 했습니다.

내 생각

캔버스 기능은 UI가 아니라 좌표계 문제였습니다. 처음부터 “저장 좌표는 캔버스 기준, 표시 좌표는 화면 기준”이라는 원칙을 세우니 이후 노트 이동, 생성, 서버 저장까지 같은 규칙으로 이어갈 수 있었습니다.

처음 열 때는 콘텐츠가 보이도록 맞췄다

캔버스가 무한하다는 것은 사용자가 어디에 있는지 잃어버리기 쉽다는 뜻이기도 합니다. 그래서 로딩 후에는 노트들이 보이도록 `fitToContent`를 호출해 초기 위치를 잡았습니다.

func load(viewSize: CGSize) async {
    isLoading = true
    defer { isLoading = false }

    if AuthViewModel.isGuestMode {
        notes = DummyData.notes
        fitToContent(viewSize: viewSize)
        return
    }

    currentUser = try await APIService.shared.fetchCurrentUser()
    workspaceMembersList = try await APIService.shared.fetchWorkspaceUsers(id: workspaceId)
    notes = try await APIService.shared.fetchNotes(workspaceId: workspaceId)
    fitToContent(viewSize: viewSize)
}

사용자가 빈 화면을 보면 데이터가 없는지, 로딩이 실패했는지, 아니면 위치가 멀리 떨어져 있는지 알 수 없습니다. 그래서 캔버스 앱일수록 첫 진입 위치를 잘 잡는 게 중요하다고 생각했습니다.

마무리 생각

5일차는 ON-IT이 앱 안에서 진짜 협업 도구처럼 느껴지기 시작한 날이었습니다. 무한 캔버스는 겉으로는 자유로운 공간이지만, 내부적으로는 좌표와 제스처 규칙이 엄격해야 한다는 걸 많이 배웠습니다.
