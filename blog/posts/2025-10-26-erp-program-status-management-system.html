🚀 Intu ERP 시스템 개발 일지: 프로그램 상태 관리 시스템 개선

NestJS Next.js TypeScript 상태관리 시간대처리 실시간계산 하위호환성 UI/UX

최근 17개의 커밋을 통해 Intu ERP 시스템의 프로그램 상태 관리 시스템을 대폭 개선했습니다. 이번 업데이트에서는 한국 시간대 기반의 실시간 상태 계산, 하위 호환성을 보장하는 상태 체계 통합, 그리고 사용자 경험 개선에 중점을 두었습니다.

기존의 UTC 기반 시간 관리에서 한국 시간대(Asia/Seoul)를 기준으로 한 정확한 상태 계산 시스템을 구현하고, 실시간으로 상태를 계산하여 사용자에게 표시하는 기능을 추가했습니다. 또한 기존 데이터와의 호환성을 보장하면서 새로운 상태 체계를 도입했습니다.

🕐 1. 한국 시간대 기반 상태 관리 시스템

문제 상황

  • 기존 UTC 기반 시간 관리로 인한 상태 계산 오류
  • 한국 사용자에게 부정확한 시간 정보 제공
  • 프론트엔드와 백엔드 간의 시간대 처리 불일치

해결 과정

1.1 새로운 상태 체계 도입

ProgramStatus 열거형 확장:

// backend/src/programs/enums/program-status.enum.ts export enum ProgramStatus { // 기존 상태들 (하위 호환성) DRAFT = 'draft', OPEN = 'open', CLOSED = 'closed', ONGOING = 'ongoing', COMPLETED = 'completed', ARCHIVED = 'archived', // 새로운 상태들 BEFORE_APPLICATION = 'before_application', APPLICATION_OPEN = 'application_open', IN_PROGRESS = 'in_progress', }

1.2 실시간 상태 계산 로직 구현

한국 시간대 기반 상태 계산 함수:

// backend/src/programs/utils/program-status.util.ts export function calculateProgramStatus( applyStart: string, applyEnd: string, programStart: string, programEnd: string ): 'draft' | 'open' | 'closed' | 'ongoing' | 'completed' | 'before_application' | 'application_open' | 'in_progress' { // 한국 시간 기준 현재 시간 const now = new Date(new Date().toLocaleString('en-US', { timeZone: 'Asia/Seoul' })); const applyStartDate = new Date(applyStart); const applyEndDate = new Date(applyEnd); const programStartDate = new Date(programStart); const programEndDate = new Date(programEnd); // 신청 기간 전 if (now < applyStartDate) { return 'before_application'; } // 신청 기간 중 if (now >= applyStartDate && now <= applyEndDate) { return 'application_open'; } // 신청 기간 종료 후, 활동 시작 전 if (now > applyEndDate && programStartDate && now < programStartDate) { return 'in_progress'; } // 활동 기간 중 if (programStartDate && programEndDate && now >= programStartDate && now <= programEndDate) { return 'in_progress'; } // 활동 종료 후 if (programEndDate && now > programEndDate) { return 'completed'; } return 'in_progress'; }

1.3 한국 시간대 유틸리티 함수 구현

시간 변환 유틸리티 함수들:

// backend/src/common/utils/time.util.ts // 한국 시간대 기준으로 현재 날짜를 YYYY-MM-DD 형식으로 반환 export function getKoreanDateString(date?: Date): string { const now = date || new Date(); const koreanTime = new Date(now.getTime() + (9 * 60 * 60 * 1000)); // UTC+9 return koreanTime.toISOString().split('T')[0]; } // 한국 날짜시간 문자열을 UTC Date 객체로 변환 export function koreanDateTimeStringToUTC(dateTimeString: string): Date { const koreanDate = new Date(dateTimeString); return new Date(koreanDate.getTime() - (9 * 60 * 60 * 1000)); // UTC로 변환 }

🔄 3. 하위 호환성 보장

문제 상황

  • 기존 데이터와 새로운 상태 체계 간의 호환성 문제
  • 기존 상태를 새로운 상태로 변환하는 로직 필요
  • 데이터 마이그레이션 없이 점진적 전환 필요

해결 과정

3.1 백엔드 상태 변환 로직

기존 상태를 새로운 상태로 변환:

// backend/src/programs/programs.service.ts private calculateProgramStatus(program: Program): ProgramStatus { const now = new Date(new Date().toLocaleString('en-US', { timeZone: 'Asia/Seoul' })); // 기존 상태가 있으면 새로운 상태로 변환 if (program.status === ProgramStatus.OPEN) { return ProgramStatus.APPLICATION_OPEN; } if (program.status === ProgramStatus.CLOSED) { return ProgramStatus.IN_PROGRESS; } if (program.status === ProgramStatus.ONGOING) { return ProgramStatus.IN_PROGRESS; } // 날짜 기반 상태 계산 if (program.applyStart && program.applyEnd && program.programStart && program.programEnd) { return calculateProgramStatus( program.applyStart, program.applyEnd, program.programStart, program.programEnd ); } return program.status; }

3.2 프론트엔드 상태 매핑

상태 라벨 매핑 함수:

// frontend/src/utils/status.util.ts export const getStatusLabel = (status: string): string => { const statusLabels: Record<string, string> = { 'draft': '초안', 'open': '신청 가능', 'closed': '신청 마감', 'ongoing': '진행 중', 'completed': '완료', 'archived': '보관', 'before_application': '신청 전', 'application_open': '신청 가능', 'in_progress': '진행 중', }; return statusLabels[status] || status; };

✅ 결과

기존 데이터와 새로운 상태 체계 간의 완벽한 호환성 보장

⚛️ 5. Next.js 호환성 개선

문제 상황

  • Next.js 15의 Suspense boundary 요구사항 미준수

  • 클라이언트 컴포넌트와 서버 컴포넌트 분리 필요

📝 결론

이번 업데이트를 통해 Intu ERP 시스템의 프로그램 상태 관리가 크게 개선되었습니다. 한국 시간대 기반의 정확한 상태 계산, 실시간 UI 업데이트, 그리고 하위 호환성 보장을 통해 사용자 경험이 향상되었습니다.
