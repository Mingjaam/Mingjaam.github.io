ON-IT 앱버전 개발일기 10일차: App Store 배포와 최종 정리

2026년 6월 9일
프론트엔드 프로젝트 학습
App Store Xcode 배포 TestFlight ON-IT

10일차에는 기능 구현보다 배포 준비에 집중했습니다. 앱은 시뮬레이터에서 잘 돌아가는 것과 App Store에 올라가는 것이 완전히 다른 단계라는 걸 다시 느꼈습니다.

프로젝트 설정을 배포 기준으로 정리했다

배포를 위해 가장 먼저 확인한 것은 Bundle Identifier, Signing Team, 버전, 빌드 번호였습니다. 개발 중에는 대충 지나갈 수 있지만, App Store Connect에서는 이 값들이 앱을 식별하는 기준이 됩니다.

DEVELOPMENT_TEAM = 89KZLFUW5B;
PRODUCT_BUNDLE_IDENTIFIER = "com.mingjam.on-it";
INFOPLIST_KEY_CFBundleDisplayName = "ON-IT";
INFOPLIST_KEY_LSApplicationCategoryType = "public.app-category.utilities";
MARKETING_VERSION = 1.0.5;
CURRENT_PROJECT_VERSION = 5;

버전은 사용자가 보는 앱 버전이고, 빌드 번호는 업로드마다 증가해야 하는 내부 번호입니다. 처음에는 둘을 비슷하게 생각했지만, 배포 과정에서는 역할이 다릅니다. 그래서 `1.0.5`와 build `5`를 명확히 맞춰두었습니다.

앱 아이콘과 로그인 첫인상도 같이 손봤다

App Store에 올릴 앱은 기능만 있어서는 부족합니다. 홈 화면에 보이는 아이콘, 로그인 화면의 첫 인상, 앱 이름이 모두 같은 제품처럼 보여야 합니다. 그래서 AppIcon asset과 앱 메인 아이콘 이미지를 프로젝트에 추가했습니다.

VStack(spacing: 18) {
    Image("ic_appmain")
        .resizable()
        .scaledToFit()
        .frame(width: 96, height: 96)
        .clipShape(RoundedRectangle(cornerRadius: 22, style: .continuous))

    Text("on-it")
        .font(.largeTitle.bold())

    Text("생각을 캔버스에 모으고, 팀과 바로 나누세요.")
        .font(.subheadline)
        .foregroundStyle(.secondary)
}

개발 중에는 로그인 화면을 기능 확인용으로만 봤는데, 배포를 앞두니 이 화면이 사용자가 처음 만나는 ON-IT이라는 생각이 들었습니다. 그래서 앱 아이콘과 브랜드 이름을 분명히 보이게 했습니다.

내 생각

제품을 배포한다는 건 코드를 완성하는 것과 조금 달랐습니다. 코드가 맞는지뿐 아니라, 사용자가 설치했을 때 “이게 어떤 앱인지” 바로 알아볼 수 있는지도 중요했습니다.

Archive부터 App Store Connect까지

최종 빌드는 Xcode에서 실제 디바이스 타깃을 기준으로 Archive를 만들고, 자동 signing이 올바른 팀으로 잡혔는지 확인한 뒤 App Store Connect로 업로드했습니다. 업로드 후에는 TestFlight에서 빌드가 처리되는지 확인하고, 앱 정보와 스크린샷, 설명, 개인정보 처리 항목을 맞춰 심사 제출 흐름까지 진행했습니다.

배포 체크리스트
1. Bundle ID: com.mingjam.on-it
2. Signing Team: 89KZLFUW5B
3. Version / Build: 1.0.5 / 5
4. App Icon asset 확인
5. Xcode Archive 생성
6. App Store Connect 업로드
7. TestFlight 빌드 처리 확인
8. 앱 정보, 스크린샷, 개인정보 항목 입력
9. 심사 제출 및 배포 완료

마지막에는 홍보 영상 쪽에서도 App Store QR과 웹사이트 QR이 보이도록 엔딩 구성을 맞췄습니다. 앱을 만들고 끝내는 것이 아니라, 사용자가 실제로 설치할 수 있는 길까지 이어줘야 한다고 생각했습니다.

그리고 2026년 6월 9일 기준으로 ON-IT 앱 버전 배포를 완료했습니다. 웹으로 시작했던 ON-IT이 iOS 앱으로도 설치 가능한 형태가 되었다는 점에서, 이번 10일 작업의 목표가 분명히 마무리된 날이었습니다.

마무리 생각

10일 동안 ON-IT을 앱으로 옮기면서 인증, 프로젝트 목록, 무한 캔버스, 실시간 채팅, 설정, 배포까지 한 번에 이어 보았습니다. 가장 크게 느낀 건 앱 개발은 기능을 “옮기는 일”이 아니라, 사용자가 손에 들고 쓰는 환경에 맞게 제품의 흐름을 다시 설계하는 일이라는 점입니다.
