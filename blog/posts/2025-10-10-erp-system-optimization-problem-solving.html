NestJS Next.js TypeScript Docker Nginx 권한관리 이미지처리 문제해결

최근 너무 바빴다. 일정도 많았고 개발을 마무리해야 하는 단계에서 블로그 올리기 힘들었다. 프로젝트 마감이 다가오면서 더욱 집중해야 할 시점이었고, 매일매일 새로운 문제들이 쏟아져 나오는 상황에서 블로그 작성까지 신경 쓸 여유가 없었다.

하지만 몇 주 남지 않은 데드라인을 지키기 위해 더 열심히 하겠다는 마음으로, 오늘 하루 동안 해결한 문제들과 개선사항들을 정리해보려고 한다. ERP 시스템의 핵심 기능들을 안정화시키고, 사용자 경험을 개선하는 과정에서 많은 것을 배웠다.

🔧 1. 직원 권한 문제 해결

문제 상황

  • 직원(staff)이 불량회원 관리 페이지에 접근할 수 없음
  • 페이지가 계속 로딩 상태로 표시됨
  • 운영자와 직원의 권한이 동일해야 함

해결 과정

1.1 백엔드 권한 설정 수정

UserReportsController 수정:

// backend/src/user-reports/user-reports.controller.ts
@Controller('user-reports')
@UseGuards(JwtAuthGuard, RolesGuard)
export class UserReportsController {
  @Get()
  @Roles(UserRole.ADMIN, UserRole.OPERATOR, UserRole.STAFF) // STAFF 추가
  async findAll(@Request() req) {
    return this.userReportsService.findAll(req.user);
  }

  @Post()
  @Roles(UserRole.ADMIN, UserRole.OPERATOR, UserRole.STAFF) // STAFF 추가
  async create(@Body() createUserReportDto: CreateUserReportDto, @Request() req) {
    return this.userReportsService.create(createUserReportDto, req.user);
  }
}

UserReportsService 수정:

// backend/src/user-reports/user-reports.service.ts
async findAll(user: User) {
  if (user.role === UserRole.ADMIN || user.role === UserRole.OPERATOR || user.role === UserRole.STAFF) {
    // 모든 마을의 신고를 볼 수 있음
    return this.userReportRepository.find({
      relations: ['reporter', 'reportedUser', 'reviewer'],
      order: { createdAt: 'DESC' },
    });
  }
  // ...
}

🗑️ 2. 대시보드 삭제된 프로그램 표시 문제 해결

문제 상황

  • 대시보드에서 삭제된 프로그램(isActive: false)이 최근 프로그램으로 표시됨
  • 활성화된 프로그램만 표시되어야 함

해결 과정

DashboardService 수정:

// backend/src/dashboard/dashboard.service.ts
private async getRecentPrograms(limit = 5, organizationId?: string) {
  let programs;
  
  if (!organizationId) {
    programs = await this.programRepository.find({
      where: { isActive: true }, // isActive 필터 추가
      relations: ['organizer', 'applications'],
      order: { createdAt: 'DESC' },
      take: limit,
    });
  } else {
    programs = await this.programRepository.find({
      where: { organizerId: organizationId, isActive: true }, // isActive 필터 추가
      relations: ['organizer', 'applications'],
      order: { createdAt: 'DESC' },
      take: limit,
    });
  }
  // ...
}

📸 3. 이미지 업로드 및 표시 문제 해결

문제 상황

  1. 이미지 업로드는 되지만 화면에 표시되지 않음
  2. 400 Bad Request 에러 발생
  3. Next.js Image 컴포넌트가 외부 도메인을 차단

해결 과정

3.1 업로드 폴더 위치 변경

Docker Compose 설정 수정:

# docker-compose.prod.yml
backend:
  volumes:
    - ../uploads:/app/uploads:rw  # ./uploads → ../uploads

nginx:
  volumes:
    - ../uploads:/var/www/uploads:rw  # ./uploads → ../uploads

폴더 구조 변경:

nuvio/
├── uploads/          # ← 새로운 위치
│   └── images/
└── intu_erp/
    ├── backend/
    ├── frontend/
    └── docker-compose.prod.yml

3.2 이미지 URL 절대 경로 변환

ImageUpload 컴포넌트 수정:

// frontend/src/components/ui/image-upload.tsx
const data = await response.json();
const { url } = data;

// 상대 경로를 절대 URL로 변환
const fullUrl = url.startsWith('http') ? url : `https://nuvio.kr${url}`;

setPreview(fullUrl);
onChange(fullUrl);

3.3 Next.js 이미지 도메인 설정

next.config.ts 수정:

// frontend/next.config.ts
const nextConfig: NextConfig = {
  // ... 기존 설정
  images: {
    remotePatterns: [
      {
        protocol: 'https',
        hostname: 'nuvio.kr',
        port: '',
        pathname: '/uploads/images/**',
      },
      {
        protocol: 'https',
        hostname: 'www.nuvio.kr',
        port: '',
        pathname: '/uploads/images/**',
      },
    ],
  },
};

3.4 Nginx 이미지 서빙 설정

nginx.conf 수정:

# nginx/nginx.conf
location /uploads/ {
    alias /var/www/uploads/;
    expires 1y;
    add_header Cache-Control "public, immutable";
    
    # 이미지 파일 MIME 타입 설정
    location ~* \.(jpg|jpeg|png|gif|webp|svg)$ {
        add_header Content-Type image/jpeg;
        add_header Content-Type image/png;
        add_header Content-Type image/gif;
        add_header Content-Type image/webp;
        add_header Content-Type image/svg+xml;
    }
}

🔧 4. 프로그램 생성 권한 문제 해결

문제 상황

  • 직원이 프로그램 생성 시 403 Forbidden 에러 발생
  • 컨트롤러에는 권한이 있지만 서비스 로직에서 누락

해결 과정

ProgramsService 수정:

// backend/src/programs/programs.service.ts
async create(createProgramDto: CreateProgramDto, user: User): Promise {
  // 권한 확인: 관리자, 운영자, 직원만 프로그램 생성 가능
  if (user.role !== UserRole.ADMIN && user.role !== UserRole.OPERATOR && user.role !== UserRole.STAFF) {
    throw new ForbiddenException('프로그램 생성 권한이 없습니다.');
  }
  // ...
}

✅ 결과

직원이 프로그램을 정상적으로 생성할 수 있음

📊 6. 문제 해결 과정에서 배운 점

1. 권한 관리의 중요성

  • 프론트엔드와 백엔드 모두에서 일관된 권한 체크 필요
  • 역할 기반 접근 제어(RBAC) 구현 시 모든 레이어에서 검증

2. 이미지 처리의 복잡성

  • 상대 경로 vs 절대 경로 처리
  • Next.js Image 컴포넌트의 외부 도메인 제한
  • Nginx를 통한 정적 파일 서빙 설정

3. Docker 컨테이너 권한 문제

  • 파일 업로드 시 컨테이너 사용자 권한 설정
  • 볼륨 마운트 시 호스트와 컨테이너 권한 일치

4. 환경별 설정 관리

  • 개발/프로덕션 환경 분리
  • 환경 변수를 통한 설정 관리
  • Docker Compose를 통한 환경 구성
