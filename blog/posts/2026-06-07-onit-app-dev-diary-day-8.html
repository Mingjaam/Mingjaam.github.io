ON-IT 앱버전 개발일기 8일차: 채팅, 멤버, 초대 링크 UX 정리하기

2026년 6월 7일
프론트엔드 프로젝트
채팅 멤버관리 초대링크 SwiftUI 협업UX

8일차에는 실시간 연결 위에 실제 사용자가 만지는 협업 UX를 올렸습니다. 캔버스만 있어서는 협업이 완성되지 않고, 누가 들어와 있는지 보고 바로 대화하고 초대할 수 있어야 했습니다.

채팅과 멤버를 하나의 패널로 묶었다

모바일 화면은 좁기 때문에 채팅과 멤버 목록을 별도 화면으로 계속 밀어내면 캔버스 흐름이 끊깁니다. 그래서 캔버스 위에서 열리는 패널 안에 Chat/Members 탭을 두었습니다.

struct ChatPanelView: View {
    @ObservedObject var vm: CanvasViewModel
    @State private var selectedTab: PanelTab = .chat

    var body: some View {
        VStack(spacing: 0) {
            Picker("", selection: $selectedTab) {
                Text("채팅").tag(PanelTab.chat)
                Text("멤버").tag(PanelTab.members)
            }
            .pickerStyle(.segmented)
            .padding()

            if selectedTab == .chat {
                ChatTab(vm: vm)
            } else {
                MembersTab(vm: vm)
            }
        }
    }
}

이렇게 한 이유는 사용자가 캔버스 작업 중에 “대화하기”와 “누가 있는지 보기”를 같은 맥락에서 하기 때문입니다. 협업 기능을 메뉴 깊숙이 넣으면 기능은 있어도 잘 쓰이지 않을 것 같았습니다.

채팅 히스토리는 이름 매핑까지 같이 처리했다

채팅 메시지는 단순히 텍스트만 보여주면 부족합니다. 누가 보냈는지, 내가 보낸 메시지인지, 시간은 언제인지가 함께 보여야 대화가 자연스럽습니다. 그래서 히스토리를 불러올 때 워크스페이스 멤버 목록과 매핑했습니다.

func loadChatHistory() async {
    do {
        let history = try await APIService.shared.fetchChatHistory(workspaceId: workspaceId)
        let nameMap = Dictionary(uniqueKeysWithValues: workspaceMembersList.map {
            ($0.userId, $0.name)
        })

        chatMessages = history.map { item in
            ChatMessage(
                id: item.id,
                senderId: item.senderId,
                senderName: item.senderId == currentUser?.userId ? "나" : (nameMap[item.senderId] ?? "알 수 없음"),
                message: item.message,
                createdAt: item.createdAt
            )
        }
    } catch {
        errorMessage = error.localizedDescription
    }
}

내 메시지를 “나”로 바꾼 것도 작은 UX입니다. 이름을 그대로 보여줘도 기능적으로는 맞지만, 채팅 앱에서 내 메시지를 바로 구분하는 게 훨씬 자연스럽습니다.

내 생각

협업 도구에서는 데이터가 정확한 것도 중요하지만, 사람이 빨리 이해할 수 있게 바꾸는 과정도 중요합니다. senderId는 서버에는 좋은 값이지만 사용자에게는 이름과 방향성이 더 좋은 정보였습니다.

초대 링크는 생성 후 바로 복사되게 했다

워크스페이스는 혼자 쓰는 공간이 아니기 때문에 초대가 쉬워야 합니다. 멤버 탭에서 초대 링크를 만들고 바로 복사할 수 있게 했습니다.

func generateInviteLink() async {
    do {
        inviteLink = try await APIService.shared.generateInviteLink(workspaceId: workspaceId).link
    } catch {
        errorMessage = error.localizedDescription
    }
}

Button {
    if let inviteLink {
        UIPasteboard.general.string = inviteLink
    }
} label: {
    Label("복사", systemImage: "doc.on.doc")
}

초대 링크를 보여주기만 하면 사용자는 다시 길게 누르거나 선택해야 합니다. 앱에서는 링크 생성의 다음 행동이 거의 항상 공유이기 때문에, 복사 버튼을 바로 붙이는 것이 맞다고 봤습니다.

마무리 생각

8일차에는 기능을 만드는 것보다 기능 사이의 거리를 줄이는 데 집중했습니다. 채팅, 멤버, 초대는 각각 따로 보면 작은 기능이지만, 캔버스와 붙어 있을 때 협업 도구다운 느낌을 만듭니다.
