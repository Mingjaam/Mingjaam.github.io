🎯 프로젝트 개요

Fashion-MNIST 데이터셋을 활용하여 기본 CNN 모델의 성능을 향상시키는 프로젝트입니다. 모델 구조 개선, 정규화 기법 적용, 학습 최적화 등을 통해 테스트 정확도를 향상시키는 과정을 기록합니다.

1. 서론

1.1 Fashion-MNIST 데이터셋 소개

Fashion-MNIST는 28×28 크기의 흑백 의류 이미지 데이터셋으로, 총 10개의 클래스(티셔츠, 바지, 풀오버, 드레스, 코트, 샌달, 셔츠, 스니커즈, 가방, 앵클 부츠)로 구성되어 있다. 학습 데이터 60,000개, 테스트 데이터 10,000개로 구성되어 있으며, 각 픽셀 값은 0-255 범위의 정수이다.

1.2 프로젝트 목적

본 프로젝트의 목적은 기본 제공된 Fashion-MNIST 분류 모델의 구조와 하이퍼파라미터를 개선하여 테스트 정확도를 향상시키는 것이다. 베이스라인 모델의 성능을 분석하고, 다양한 개선 방법을 실험하여 최적의 모델을 찾아내는 과정을 통해 딥러닝 모델 최적화 방법을 학습한다.

2. 베이스라인 모델

2.1 모델 구조

베이스라인 모델은 다음과 같은 간단한 CNN 구조로 구성되어 있다:

# 베이스라인 모델 구조 Input (28×28×1) ↓ Conv2D(32, 3×3, activation='relu') ↓ MaxPooling2D(2×2) ↓ Conv2D(64, 3×3, activation='relu') ↓ MaxPooling2D(2×2) ↓ Flatten() ↓ Dense(64, activation='relu') ↓ Dense(10, activation='softmax')

📝 주요 특징

  • 2개의 Conv2D 레이어 (32, 64 필터)
  • 각 Conv 레이어 후 MaxPooling2D 적용
  • 1개의 Dense 레이어 (64 뉴런)
  • 활성화 함수: ReLU (은닉층), Softmax (출력층)

2.2 학습 설정

항목 값
Optimizer Adam (기본 학습률 0.001)
Loss Function Categorical Crossentropy
Epochs 30
Batch Size 512
Validation Split 0.2

3.2 실험 모델 설정

모델 A: 베이스라인 모델

  • 구조: Conv(32) → Conv(64) → Dense(64)
  • 정규화: 없음
  • Optimizer: Adam (고정 학습률)

모델 B: 용량 증가 모델

  • 구조: Conv(64) → Conv(128) → Conv(128) → Dense(128) → Dense(64)
  • 정규화: Batch Normalization, Dropout(0.2, 0.3)
  • Optimizer: Adam (고정 학습률)

모델 C: 최종 개선 모델

  • 구조: Conv(64) → Conv(128) → Conv(128) → Dense(128) → Dense(64)
  • 정규화: Batch Normalization, Dropout(0.2, 0.3)
  • Optimizer: Adam (학습률 스케줄러 적용)
  • Padding: 모든 Conv 레이어에 'same' 적용

6. 결론

본 프로젝트를 통해 다음과 같은 점을 학습하였다:

📚 주요 학습 내용

  1. 모델 구조 개선의 중요성
    적절한 모델 용량 증가가 성능 향상에 직접적인 영향을 미치며, 단순히 크게 만드는 것이 아니라 정규화와 함께 균형 있게 설계해야 함
  2. 정규화 기법의 효과
    Batch Normalization과 Dropout을 적절히 조합하면 과적합을 방지하면서도 학습 능력을 유지할 수 있음
  3. 학습 최적화의 필요성
    학습률 스케줄러를 통해 더 정밀한 최적화가 가능하며, 초기 학습과 후반 학습의 균형이 중요함
  4. 실험적 접근의 중요성
    이론적 가설을 실제 실험으로 검증하는 과정에서 더 나은 이해를 얻을 수 있으며, 체계적인 실험 설계와 결과 분석이 필수적임

7.2 데이터셋별 상세 변경 사항

7.2.1 MNIST (손글씨 숫자)

변경 필요도: 매우 낮음

# 변경 사항 from tensorflow.keras.datasets import mnist # fashion_mnist → mnist # 데이터 로드 (train_images, train_labels), (test_images, test_labels) = mnist.load_data() # 모델 구조: 변경 없음 (28×28×1, 10 클래스 동일) # input_shape=(28, 28, 1) 유지 # 출력 레이어: Dense(10, activation='softmax') 유지

특징: Fashion-MNIST와 동일한 구조 사용 가능하며, 가장 간단한 데이터셋으로 빠른 학습 가능

7.2.2 CIFAR-10 (컬러 이미지)

변경 필요도: 높음

# 변경 사항 from tensorflow.keras.datasets import cifar10 # 데이터 로드 (train_images, train_labels), (test_images, test_labels) = cifar10.load_data() # 1. 입력 shape 변경 (채널 수 변경) # 기존: input_shape=(28, 28, 1) # 변경: input_shape=(32, 32, 3) # RGB 3채널 # 2. 모델 구조 조정 model = tf.keras.models.Sequential() # 첫 번째 Conv 레이어 model.add(tf.keras.layers.Conv2D(64, (3, 3), activation='relu', input_shape=(32, 32, 3), # ← 변경 padding='same')) # 3. 데이터 전처리 (이미 정규화되어 있을 수 있음) # train_images = train_images / 255.0 # 필요시 확인 # 4. 출력 레이어: 변경 없음 (10 클래스 동일) model.add(tf.keras.layers.Dense(10, activation='softmax'))

💡 추가 고려사항

  • 이미지 크기가 32×32로 더 큼 → 더 많은 파라미터 필요
  • 컬러 이미지로 복잡도 증가 → 모델 용량 증가 고려
  • 데이터 증강 효과가 더 클 수 있음

7.2.3 CIFAR-100 (100개 클래스)

변경 필요도: 매우 높음

# 변경 사항 from tensorflow.keras.datasets import cifar100 # 데이터 로드 (train_images, train_labels), (test_images, test_labels) = cifar100.load_data() # 1. 입력 shape 변경 # input_shape=(32, 32, 3) # CIFAR-10과 동일 # 2. 출력 레이어 변경 (클래스 수 변경) # 기존: Dense(10, activation='softmax') # 변경: Dense(100, activation='softmax') # 100개 클래스 # 3. 모델 용량 증가 필요 # - 더 많은 필터 수 (128 → 256 등) # - 더 깊은 네트워크 (레이어 추가) # - 더 큰 Dense 레이어 (128 → 256 등) # 4. 정규화 강화 필요 # - Dropout 비율 증가 (0.3 → 0.4, 0.5) # - L2 정규화 추가 고려 # - 데이터 증강 필수 # 5. 학습 설정 조정 # - Epoch 수 증가 (30 → 50, 100) # - 학습률 스케줄러 필수 # - Early Stopping 필수

⚠️ 추가 고려사항

  • 클래스 수가 10배 증가 → 모델 복잡도 대폭 증가 필요
  • 과적합 위험 높음 → 강한 정규화 필수
  • 학습 시간 증가 예상

7.2.4 KMNIST (일본 고문서)

변경 필요도: 매우 낮음

7.6 체크리스트

다른 데이터셋 적용 시 확인 사항:

  • 이미지 크기 확인 및 input_shape 조정
  • 채널 수 확인 (1: 흑백, 3: 컬러)
  • 클래스 수 확인 및 출력 레이어 조정
  • 데이터 정규화 확인 (0-255 → 0-1)
  • 레이블 인코딩 확인 (원-핫 인코딩)
  • 모델 용량 조정 (필터 수, 레이어 수)
