CSS
HTML이 뼈대라면 CSS는 옷이에요

색상, 크기, 여백, 위치 등 시각적인 스타일을 담당해요.
HTML과 별도의 .css 파일로 분리해서 작성해요.

<!-- head 안에 추가 -->
<link rel="stylesheet" href="style.css" />

CSS 기본 문법

/* 선택자 { 속성: 값; } */

h1 {
  color: red;
  font-size: 32px;
  font-weight: bold;
}

.container {
  background-color: #f5f5f5;
  padding: 20px;
}
→

선택자

어떤 태그에 스타일을 줄지 지정

→

속성

무엇을 바꿀지 — color, font-size ...

→

값

어떻게 바꿀지 — red, 32px ...

선택자 3종류

태그 선택자

h1 {
  color: red;
}

모든 <h1> 태그에 적용

class 선택자

.box {
  padding: 20px;
}

class="box" 인 요소에 적용 — 가장 많이 써요

id 선택자

#header {
  background: blue;
}

id="header" 인 요소에 적용

Flexbox
요소들을 가로 · 세로로 정렬하는 레이아웃 도구

CSS 레이아웃의 핵심이에요. display: flex 하나로 시작해요.

.container {
  display: flex;            /* Flexbox 시작 */
  flex-direction: row;      /* 방향: row(가로) | column(세로) */
  justify-content: center;  /* 주축 정렬 */
  align-items: center;      /* 교차축 정렬 */
  gap: 16px;                /* 자식 요소 간격 */
}

React 미리보기

React, 왜 배워야 할까?

HTML/CSS/JS만으로는 한계가 있어요 — React가 그 문제를 해결해요

HTML/CSS/JS vs React

😓 Vanilla JS (기존 방식)

// 화면을 직접 찾아서 수정
let count = 0

function add() {
  count++
  document.getElementById('num')
    .innerText = count
  // 요소가 늘어날수록
  // 코드가 뒤엉켜요
}
  • DOM을 직접 찾아서 직접 바꿔야 함
  • 코드가 복잡해질수록 관리 어려움
  • 같은 UI를 여러 곳에 복붙해야 함

✅ React (새로운 방식)

// 데이터만 바꾸면 끝!
const [count, setCount]
  = useState(0)

<button onClick={() =>
  setCount(count + 1)}>
  {count}
</button>
// React가 화면을 알아서 업데이트
  • 데이터(state)만 바꾸면 화면 자동 업데이트
  • 컴포넌트로 UI를 재사용 가능하게 분리
  • 대규모 앱도 체계적으로 관리 가능

React를 배우면 뭐가 좋나요?

💼

취업에 유리해요

국내외 스타트업, 대기업 대부분이 React를 사용해요. 프론트엔드 채용공고의 80% 이상이 React 요구

🧩

재사용이 쉬워요

버튼, 카드, 모달 등 UI를 컴포넌트로 만들어두면 어디서든 가져다 쓸 수 있어요

⚡

생산성이 높아요

복잡한 UI도 상태(state) 하나만 바꾸면 화면이 자동으로 업데이트되어 빠르게 개발할 수 있어요

🚀

확장이 쉬워요

React를 알면 Next.js(풀스택), React Native(앱 개발)로 자연스럽게 확장할 수 있어요
