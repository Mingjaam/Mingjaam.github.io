Intro

State와 Props

React 컴포넌트에서 데이터를 다루는 두 가지 핵심 개념입니다.

State

컴포넌트가 스스로 기억하는 값입니다.
State가 바뀌면 React가 화면을 자동으로 다시 그립니다.

예: 카운터 숫자, 좋아요 눌렸는지 여부, 입력창에 타이핑한 내용

Props

부모가 자식 컴포넌트에 전달하는 값입니다.
자식은 props를 읽기만 할 수 있고, 직접 변경할 수 없습니다.

예: 부모가 자식에게 숫자, 문자열, 함수를 내려보냄

항목 State Props
누가 만드나 컴포넌트 자신이 useState로 생성 부모가 만들어서 자식에게 전달
변경 가능? 가능 — setter 함수로만 변경 불가능 — 자식은 읽기만 가능
바뀌면? 컴포넌트가 다시 렌더링됨 부모 state가 바뀌면 새 값이 내려옴
선언 위치 컴포넌트 함수 내부 컴포넌트 함수의 매개변수
State 변경 주의: count = count + 1 처럼 직접 대입하면 화면이 바뀌지 않습니다. 반드시 setCount(count + 1) 처럼 setter 함수를 사용해야 합니다.
Example 1

카운터

버튼을 클릭하면 숫자가 1씩 올라갑니다. State만 사용하는 기본 예제입니다.

State

Counter.jsx  +  App.jsx

Counter.jsx 컴포넌트 
import { useState } from 'react';

function Counter() {
  // 현재값      변경함수        초기값
  const [count, setCount] = useState(0);

  return (
    <div>
      <h2>현재 숫자: {count}</h2>

      <button onClick={() => setCount(count + 1)}>
        +1 증가
      </button>

      <button onClick={() => setCount(0)}>
        초기화
      </button>
    </div>
  );
}

export default Counter;
App.jsx 메인 앱 
import Counter from './Counter';

function App() {
  return (
    <div>
      <h1>카운터 앱</h1>

      {/* props 없이 그냥 가져다 씁니다 */}
      <Counter />
    </div>
  );
}

export default App;
실행 결과 미리보기
  • 01 useState(0) — 초기값 0으로 state 생성. count는 현재 값, setCount는 값을 바꾸는 함수
  • 02 {"{count}"} — JSX에서 중괄호로 변수를 출력. count가 바뀌면 이 부분도 자동으로 업데이트됨
  • 03 onClick={() => setCount(count + 1)} — 버튼 클릭 시 setCount 호출. 화살표 함수로 감싸야 클릭할 때만 실행됨
  • 04 App.jsx에서 <Counter />만 가져다 씁니다. Counter 내부의 state는 Counter 스스로 관리하기 때문에 props 전달 불필요
Example 2

좋아요 버튼

좋아요 버튼을 누르면 흰 하트(🤍)가 빨간 하트(❤️)로 바뀝니다. Boolean state와 조건부 렌더링을 Props로 전달하는 예제입니다.

State Props

LikeButton.jsx  +  App.jsx

LikeButton.jsx 컴포넌트 
// props: liked(boolean), onLike(함수)
function LikeButton({ liked, onLike }) {
  return (
    <div>
      {/* liked 값에 따라 하트 이모지가 바뀜 */}
      <div style={{ fontSize: '2.5rem' }}>
        {liked ? '❤️' : '🤍'}
      </div>

      {/* 아래 버튼을 누르면 토글됨 */}
      <button onClick={onLike}>
        좋아요
      </button>
    </div>
  );
}

// state 없음 — liked props 값에 따라
// 어떤 하트를 보여줄지 결정함
export default LikeButton;
App.jsx 메인 앱 
import { useState } from 'react';
import LikeButton from './LikeButton';

function App() {
  // false = 아직 좋아요 안 누름
  // true  = 좋아요 누름
  const [liked, setLiked] = useState(false);

  return (
    <div>
      <h1>오늘의 게시물</h1>
      <p>React는 재미있어요!</p>

      <LikeButton
        liked={liked}
        onLike={() => setLiked(!liked)}
      />
    </div>
  );
}

export default App;
실행 결과 미리보기
  • 01 useState(false) — 숫자 대신 boolean(true/false)으로 state 생성. false = 좋아요 안 누름, true = 좋아요 누름
  • 02 setLiked(!liked) — !liked는 현재 값의 반대. true면 false, false면 true. 이것이 토글(toggle) 패턴
  • 03 {liked ? '❤️' : '🤍'} — liked가 true면 빨간 하트, false면 흰 하트. props 값에 따라 다른 내용을 보여주는 조건부 렌더링
  • 04 하트(표시)와 버튼(클릭)을 분리. 버튼의 onClick={"{onLike}"}는 부모에서 받은 함수를 그대로 연결
왜 state를 부모(App)에 둘까요?
LikeButton은 화면 표시만 담당합니다. 상태 관리는 App이 합니다. 이렇게 역할을 분리하면 LikeButton을 다른 곳에서도 재사용하기 쉬워집니다. 이것을 State 끌어올리기(Lifting State Up)라고 합니다.
