React란?

배경 · 특징 · 왜 배우는가

React란 무엇인가?

React는 Meta(Facebook)가 만든 UI 빌드 라이브러리입니다. 화면을 작은 컴포넌트 단위로 쪼개어 만들고, 데이터가 바뀌면 자동으로 화면을 업데이트해 줍니다.

  • 2013년 Facebook 오픈소스 공개
  • 전 세계 1위 프론트엔드 라이브러리
  • Netflix · Airbnb · 당근마켓 · 토스 등에서 사용
  • React Native로 모바일 앱 개발도 가능
  • Next.js(풀스택)의 기반 기술

컴포넌트

UI를 재사용 가능한 조각으로 분리

어떻게가 아닌 무엇을 표현

왜 React를 배우는가?

—

생산성

컴포넌트를 레고처럼 조합해 복잡한 UI도 빠르게 개발. 한 번 만든 버튼, 카드는 어디서든 재사용 가능해요.

—

취업 경쟁력

국내외 프론트엔드 채용 공고의 80% 이상이 React를 요구해요. 사실상 업계 표준 기술입니다.

—

확장성

React 지식은 React Native(앱)와 Next.js(풀스택)로 바로 확장돼요. 하나 배우면 셋을 얻어요.

—

상태 관리의 명확함

데이터가 바뀌면 UI가 자동 갱신되는 단방향 데이터 흐름 덕분에 버그 추적이 쉬워요.

DOM이란?

DOM (Document Object Model)은 브라우저가 HTML 파일을 읽은 뒤 만드는 트리 구조의 객체 모델이에요. 우리가 쓴 HTML 코드가 실제로 브라우저 메모리 안에서 JavaScript로 접근·수정 가능한 객체들의 트리로 변환된 것입니다.

DOM이 왜 중요한가?

  • JavaScript는 DOM을 통해 화면의 내용을 읽고 바꿀 수 있어요
  • 버튼 클릭, 텍스트 변경, 새 요소 추가 등 모든 동적 기능이 DOM 조작
  • React도 결국 DOM을 다루지만, 개발자 대신 해주는 것

HTML → 브라우저 → DOM 트리

<html>
  <body>
    <h1>제목</h1>
    <p>내용</p>
  </body>
</html>

DOM 조작은 비용이 커요. 바뀔 때마다 브라우저가 레이아웃을 재계산해야 하거든요. 이 문제를 해결한 것이 React의 Virtual DOM입니다!

React vs 순수 JavaScript

순수 JS — 명령형

// DOM을 직접 찾아서 조작해야 함
const btn  = document.querySelector('button');
const span = document.querySelector('#count');
let count = 0;

btn.addEventListener('click', () => {
  count++;
  span.textContent = count; // 직접 수정
});

// 코드가 길어질수록 관리 지옥

DOM 선택 → 값 변경 → UI 업데이트를 매번 수동으로

React — 선언형

// 상태만 정의하면 UI는 자동!
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>현재: {count}</p>
      <button onClick={() => setCount(count+1)}>
        +1 증가
      </button>
    </div>
  );
}

setCount() 한 번이면 React가 알아서 화면 갱신

Virtual DOM — 왜 빠른가?

실제 DOM 방식

  • 변경마다 실제 DOM을 직접 수정
  • 브라우저가 레이아웃을 매번 재계산 (Reflow)
  • 화면을 처음부터 다시 그릴 수 있어 느림 (Repaint)

Virtual DOM 방식 (React)

  • 변경 사항을 메모리 속 가상 DOM에 먼저 반영
  • 이전 상태와 비교(diffing)해 달라진 부분만 파악
  • 실제 DOM은 최소한으로만 업데이트

React 업데이트 흐름

① State / Props 변경
↓
② Virtual DOM 새로 생성
↓
③ 이전 Virtual DOM과 비교 (Diff)
↓
④ 변경된 부분만 실제 DOM 업데이트

JSX

JavaScript 안에 HTML을 쓰는 문법 확장

JSX란?

JSX (JavaScript XML)는 JavaScript 파일 안에 HTML처럼 보이는 코드를 작성할 수 있는 문법 확장이에요. 브라우저는 JSX를 모르기 때문에 Vite(내부의 Babel)가 순수 JS로 변환해 줍니다.

// 우리가 쓰는 JSX
const el = <h1>안녕, React!</h1>;

// Babel이 변환한 결과 (순수 JS)
const el = React.createElement(
  'h1', null, '안녕, React!'
);

JSX가 있으면 좋은 이유

  • HTML 구조를 JS 안에서 직관적으로 표현
  • 변수, 로직을 {}로 바로 삽입 가능
  • 컴포넌트를 HTML 태그처럼 사용 가능
  • 에디터의 자동완성, 타입 검사 지원
!

JSX는 선택 사항이지만 React 개발자 99%가 사용해요. .jsx 확장자를 쓰는 이유가 바로 이 때문!

JSX 핵심 문법 규칙 4가지

① 하나의 루트 요소

//  루트가 두 개
return <h1>제목</h1>
        <p>내용</p>

//  Fragment로 감싸기
return (
  <>
    <h1>제목</h1>
    <p>내용</p>
  </>
)

② 모든 태그는 닫아야 함

//  닫지 않음
<img src="photo.jpg">
<br>
<input type="text">

//  자기 닫기 태그
<img src="photo.jpg" />
<br />
<input type="text" />

③ class → className

//  HTML 방식
<div class="box"></div>

//  JSX 방식
<div className="box"></div>

// for → htmlFor 도 마찬가지!
<label htmlFor="input">
  이름
</label>

④ 인라인 스타일은 객체

//  문자열
<div style="color: red; font-size: 16px">

//  객체 + camelCase
<div style={{
  color: 'red',
  fontSize: '16px',
  marginTop: '8px'
}}>

JSX에서 JavaScript 사용하기 { }

중괄호 {} 안에 모든 JS 표현식을 삽입할 수 있어요

function Profile() {
  const name    = '김철수';
  const age     = 25;
  const isAdmin = true;
  const getRole  = () => isAdmin ? '관리자' : '회원';

  return (
    <div>
      <h1>{name}</h1>              // 변수
      <p>내년 나이: {age + 1}</p>  // 계산식
      <p>{name.toUpperCase()}</p>  // 함수 호출
      <p>역할: {getRole()}</p>     // 함수 반환값
    </div>
  );
}

{} 안에 쓸 수 있는 것

  • 변수: {name}
  • 계산식: {age * 2}
  • 함수 호출: {getName()}
  • 삼항 연산자: {ok ? '예' : '아니요'}
  • && 연산자: {show && <Modal />}
  • 배열 메서드: {arr.map(...)}

{} 안에 쓸 수 없는 것

  • if문, for문 (문장은 불가)
  • 객체 직접 출력 {{name: '철수'}}
  • undefined, null → 아무것도 안 보임

컴포넌트

UI를 만드는 기본 단위

컴포넌트란?

컴포넌트는 UI를 만드는 독립적인 단위입니다. JavaScript 함수처럼 생겼고 JSX를 반환해요. 레고 블록처럼 조합해 복잡한 화면을 만들어요.

  • 이름이 대문자로 시작해야 함 (소문자면 HTML 태그로 인식)
  • JSX를 반환하는 함수
  • 재사용 가능하고 독립적
  • 다른 컴포넌트 안에 포함(중첩) 가능
  • 파일당 한 컴포넌트를 권장 (Button.jsx)
// 가장 기본적인 함수형 컴포넌트
function Hello() {
  return <h1>안녕하세요!</h1>;
}

// 화살표 함수도 OK
const Hello = () => {
  return <h1>안녕하세요!</h1>;
};

// 사용할 때: HTML 태그처럼!
function App() {
  return (
    <div>
      <Hello /> // 컴포넌트 사용
      <Hello /> // 재사용!
    </div>
  );
}

컴포넌트 완전 해부

// ① import — 필요한 것 가져오기
import { useState } from 'react';
import './Card.css';

// ② 컴포넌트 정의 (대문자 시작!)
function Card({ title, content }) {

  // ③ 로직 (훅, 함수, 계산 등)
  const [liked, setLiked] = useState(false);
  const handleLike = () => setLiked(!liked);

  // ④ return — JSX 반환
  return (
    <div className="card">
      <h2>{title}</h2>
      <p>{content}</p>
      <button onClick={handleLike}>
        {liked ? '' : ''}
      </button>
    </div>
  );
}

// ⑤ export — 외부에서 사용 가능하게
export default Card;
①

import

React 훅, CSS, 다른 컴포넌트를 파일 상단에서 불러와요

②

function 이름() {}

반드시 대문자로 시작. 소문자면 HTML 태그로 인식!

③

로직 영역

useState 같은 훅, 이벤트 핸들러 함수, 계산 로직을 여기에 작성

④

return (JSX)

화면에 보여줄 UI를 JSX로 반환. 하나의 루트 요소만!

⑤

export default

다른 파일에서 import Card from './Card' 로 사용 가능

컴포넌트 조합하기

// 각각의 독립 컴포넌트
function Header() {
  return <header> 나의 앱</header>;
}

function Footer() {
  return <footer>© 2025</footer>;
}

function Content() {
  return <main>메인 내용</main>;
}

// App에서 조합!
function App() {
  return (
    <>
      <Header />
      <Content />
      <Footer />
    </>
  );
}
export default App;

Props

부모에서 자식으로 데이터를 전달하는 방법

Props란?

Props (Properties)는 부모 컴포넌트가 자식에게 데이터를 전달하는 방식이에요. HTML 속성처럼 작성하고, 자식은 함수 매개변수로 받습니다.

부모 컴포넌트 (App)
props 전달 (읽기 전용)
자식 컴포넌트 (Greeting)
  • Props는 읽기 전용자식이 수정 불가
  • 문자열, 숫자, 배열, 객체, 함수 모두 전달 가능
  • 컴포넌트를 동적이고 재사용 가능하게 만들어줌
// 부모: HTML 속성처럼 전달
function App() {
  return (
    <Greeting
      name="철수"
      age={25}
      isAdmin={true}
    />
  );
}

// 자식: 매개변수로 받기
function Greeting(props) {
  return (
    <div>
      <h1>안녕, {props.name}!</h1>
      <p>나이: {props.age}살</p>
      <p>{props.isAdmin ? '관리자' : '회원'}</p>
    </div>
  );
}

Props 심화 — 구조 분해 & children

구조 분해 할당 — props를 더 깔끔하게

// props 통째로 받기 (번거로움)
function Card(props) {
  return <h1>{props.title}</h1>;
}

//  구조 분해 (권장!)
function Card({ title, content, color }) {
  return (
    <div style={{ background: color }}>
      <h2>{title}</h2>
      <p>{content}</p>
    </div>
  );
}

기본값(default) & children props

// 기본값 설정
function Button({
  label = '클릭',
  color = 'blue'
}) {
  return (
    <button style={{ color }}>{label}</button>
  );
}

// children: 태그 사이의 내용
function Box({ children }) {
  return (
    <div className="box">{children}</div>
  );
}

// 사용: 태그 사이가 children
<Box><p>박스 안 내용!</p></Box>

React Hooks

함수형 컴포넌트에 능력을 부여하는 특수 함수

React Hooks 한눈에 보기

Hook은 함수형 컴포넌트에서 상태 관리, 생명주기, 최적화 등의 기능을 사용할 수 있게 해주는 특수 함수에요. 이름이 항상 use로 시작하는 것이 규칙입니다.

필수 useState

상태 관리

컴포넌트 안의 데이터(상태)를 기억하고 변경할 때 사용. 값이 바뀌면 화면 자동 갱신.

필수 useEffect

부수 효과

렌더링 이후 API 호출, 타이머 설정, 구독 등 외부와 상호작용할 때 사용.

자주 사용 useRef

DOM 접근 & 값 보존

DOM 요소에 직접 접근하거나, 렌더링을 유발하지 않는 값을 저장할 때 사용.

고급 useMemo

계산 결과 캐싱

복잡한 계산 결과를 기억해 두어 불필요한 재계산을 방지하는 성능 최적화 훅.

고급 useCallback

함수 캐싱

함수를 메모이제이션하여 자식 컴포넌트에 불필요한 리렌더링을 막을 때 사용.

State & useState

컴포넌트가 기억하는 동적 데이터

State란? — Props와의 차이

State는 컴포넌트 내부에서 시간이 지나며 변할 수 있는 데이터에요. State가 바뀌면 React가 자동으로 화면을 다시 그려줍니다.

구분PropsState
출처부모 컴포넌트컴포넌트 내부
변경읽기 전용 (불변)자유롭게 변경
예시이름, 색상 전달카운터, 입력값
!

일반 변수(let count = 0)를 바꿔도 화면이 안 바뀌어요! 반드시 useState를 사용해야 React가 변화를 감지합니다.

State가 필요한 경우

  • 버튼 클릭 횟수 (카운터)
  • input에 입력한 텍스트 값
  • 모달/메뉴의 열림·닫힘 여부
  • 서버에서 받아온 데이터 목록
  • 선택된 탭이나 필터 상태
  • 로딩 / 에러 여부

useState — 구조 완전 이해

import { useState } from 'react';

function Counter() {
  //  현재 상태값   상태 변경 함수  초기값
  const [count, setCount] = useState(0);
  //                                
  // 읽기 전용    setXxx 이름 권장  첫 값

  return (
    <div>
      <p>현재: {count}</p>
      <button
        onClick={() => setCount(count + 1)}>
        +1 증가
      </button>
      <button
        onClick={() => setCount(0)}>
        리셋
      </button>
    </div>
  );
}

useState 다양한 타입 활용

문자열 State (입력 제어)

const [name, setName] = useState('');

<input
  value={name}

    setName(e.target.value)}

const [isOpen, setIsOpen] =
  useState(false);

<button
  onClick={() => setIsOpen(!isOpen)}>
  {isOpen ? '닫기' : '열기'}
</button>
{isOpen && <Modal />}

배열 State (목록 추가)

const [items, setItems] =
  useState([]);

const addItem = (text) => {
  // 스프레드로 기존 배열 복사
  setItems([...items, text]);
};

객체 State (부분 수정)

const [user, setUser] =
  useState({ name: '', age: 0 });

// 스프레드로 기존 유지 + 수정
setUser({
  ...user,
  name: '철수'

이벤트 처리

사용자의 동작에 반응하는 방법

이벤트 핸들러

function Form() {
  const [text, setText] = useState('');

  // 핸들러 함수를 따로 정의 (권장)
  const handleChange = (e) => {
    setText(e.target.value);
  };

  const handleSubmit = (e) => {
    e.preventDefault(); // 새로고침 방지
    alert(`입력: ${text}`);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={text}
        onChange={handleChange}
      />
      <button type="submit">제출</button>
    </form>
  );
}

리스트 렌더링

배열 데이터를 화면에 출력하기

map()으로 리스트 렌더링 + key

function TodoList() {
  const todos = [
    { id: 1, text: 'React 공부',   done: true },
    { id: 2, text: '운동하기',     done: false },
    { id: 3, text: '독서 30분',    done: false },
  ];

  return (
    <ul>
      {todos.map((todo) => (
        <li
          key={todo.id}   반드시 고유 key!
          style={{
            textDecoration:
              todo.done ? 'line-through' : 'none'
          }}
        >
          {todo.text}
        </li>
      ))}
    </ul>
  );
}

배열의 .map()을 사용해 각 요소를 JSX로 변환해요. 반드시 각 항목에 고유한 key 속성을 추가해야 합니다.

key가 왜 필요한가?

  • React가 어떤 항목이 추가·삭제·변경됐는지 파악하는 데 사용
  • key는 목록 내에서 고유해야 함
  • 배열 인덱스(0,1,2)를 key로 쓰면 순서 변경 시 버그 가능
  • 가능하면 데이터 고유 ID를 key로 사용!
!

key는 화면에 보이지 않아요. React 내부에서만 사용하는 식별자입니다!

useEffect

렌더링 이후 실행되는 부수 효과(Side Effect)

useEffect 완전 정복

import { useState, useEffect } from 'react';

function App() {
  const [count, setCount] = useState(0);
  const [data, setData]   = useState(null);

  // ① 마운트 시 1회만 실행 (빈 배열)
  useEffect(() => {
    fetch('/api/data')
      .then(res => res.json())
      .then(json => setData(json));
  }, []);

  // ② count 바뀔 때마다 실행
  useEffect(() => {
    document.title = `카운트: ${count}`;
  }, [count]);

  // ③ cleanup: 언마운트 시 실행
  useEffect(() => {
    const id = setInterval(() => {
      console.log('tick');
    }, 1000);
    return () => clearInterval(id); // cleanup
  }, []);
}

의존성 배열 — 핵심!

형태실행 시점
useEffect(fn)매 렌더링 후마다
useEffect(fn, [])마운트 시 단 1회
useEffect(fn, [x])x가 바뀔 때마다
return () => {}언마운트 시 cleanup

주로 쓰는 경우

  • API 데이터 불러오기 (fetch)
  • 타이머 설정 (setInterval / setTimeout)
  • document.title 변경
  • 이벤트 리스너 등록/해제
